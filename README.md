# Utopia Photo Booth

一個為校園活動而做的瀏覽器拍貼機專案。

這個專案最初是為了以社團名義參與學校音樂節擺攤而開發。當時希望攤位不只是靜態展示，而是能讓路過的人停下來互動、拍照、帶走成品，因此決定自己做一套能在平板或筆電上直接使用的拍貼機網頁。

它不是商業化機台，也不是為了展示而做的空殼作品，而是一個在真實活動需求下逐步長出來、能夠實際測試與運作的工具型專案。

## 專案背景

這個專案由學生自行規劃與開發，背景來自一次校園音樂節擺攤需求。因為希望社團攤位不只是「站著宣傳」，而是能夠透過拍照與互動把人留下來，於是開始嘗試自己做一套拍貼機系統。

整體思路很直接：

- 用前端網頁取代原生 App
- 用平板或筆電作為操作設備
- 支援外接鏡頭測試
- 套用自製相框
- 產生可下載的拍貼成品
- 讓工作人員可以在活動現場快速接手操作

它本質上是一個「從活動需求出發」的開發專案，而不是先有技術再去找用途。

## 技術說明

### 前端架構

這個專案以單一 HTML 為核心，將畫面、流程與邏輯整合在前端頁面中，降低部署難度，也方便在 GitHub Pages 這類靜態託管環境直接上線。

### 相機存取

透過瀏覽器相機能力讀取裝置相機，並支援選擇鏡頭來源。這也是整個專案最重要的基礎能力，因為實際活動現場常常會面臨：

- 筆電前鏡頭畫質不夠
- 平板鏡頭切換不直覺
- 外接鏡頭相容性不穩定

所以相機選擇邏輯與測試過程，是這個專案開發裡非常重要的一段。

### 相框套版

拍貼機最關鍵的不是只有拍照，而是「照片要能正確放進相框」。

這件事後來變成專案裡最重要的開發細節之一：

- 照片區域必須透明
- 三格的位置必須固定
- 比例不能跑掉
- 相框風格可以換，但照片洞的位置要一致

所以後續相框製作已經不是單純做圖，而是「要符合程式規格的圖」。

### 成品輸出

拍攝完後，系統會把三張照片依照固定版型與相框合成，最後輸出成一張完整圖片，方便現場下載、傳送或後續列印。

## 開發脈絡

這個專案不是一次完成的，而是在不斷試錯中迭代出來的。

一開始只是想做出能拍照的最簡單版本，後來逐步加入：

- 固定 4x6 輸出概念
- 三格版型
- 即時預覽
- 相框管理
- 工作人員模式
- 上傳照片補圖
- 外接鏡頭測試
- GitHub Pages 線上部署

很多版本的調整，並不是為了讓畫面更華麗，而是因為真實活動場景會逼著系統變得更穩定、更直覺。
